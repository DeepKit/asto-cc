# AI 问题显影方法论总览（PFM）

> **层序**：第 7 层（编号 54），实践链独立层
> **定位**：问题显影 + 承接 + 转交的协议层
> **本质**：让尚未被命名的问题场出现，并以有边界的方式接住、编码、转交
> **历史别名**：LMM / Lighthouse Marketing Methodology / 灯塔营销方法论
> **2026-06-13 canonical status**：本文为 PFM 的 AI 协作者入口 dispatcher；`SOURCE_OF_TRUTH.md` 是当前唯一真相源登记，`PUBLISH_READINESS.md` 固定公开发布阻塞项。

---

## 一句话定义

**PFM 负责让尚未被命名的问题场出现，并把它以有边界的方式接住、编码、转交。**

> **让说不清的问题，变得可说、可见、可承接。**

---

## 第一原语：问题显影

**问题显影**——把模糊不适、弱信号、矛盾表达、反复卡住、行动阻塞或结构损耗，转化为可命名、可复述、可编码、可承接的问题场的过程。

PFM 的输入不是"成熟需求"，而是未显影材料：
- 模糊不舒服
- 说不清的请求
- 反复失败的行动
- 结构性损耗
- 弱信号、承压点

PFM 的输出不是方案、诊断或成交，而是 `structured_problem_intake`：
- PFM.issue_field
- PFM.pressure_bearer
- PFM.constraint
- PFM.weak_signal
- PFM.boundary_notice
- PFM.handoff_target
- PFM.stop_condition

PFM-min 七字段只形成接入与转交记录；它不证明问题已解决，不替代 COP / TAT / ODD / 专业渠道的后续判断。

## 三大支柱

| 支柱 | 含义 |
|------|------|
| **① 问题显影** | 解决"到底是什么问题"——第一原语 |
| **② 路径降维** | 把复杂困局压成下一步可判断、可行动、可退出的路径 |
| **③ 边界承接** | 明确谁接、接到哪里、何时转交、何时停止 |

---

## 核心闭环

```
问题在哪里发生 → 问题在哪里愿意被说出 → 谁是真正承压者 → 什么表达能让问题被认出 → 如何低阻自检 → 如何边界承接 → 如何转交或停止
```

旧术语对照：寻域→寻找问题场，确路→确认路径，定人→识别承压者，灯塔→显影触发器，自测→低阻显影工具，港口→承接口，首次责任闭合→首次承接/转交记录。

## PFM 五协议

```
场域探测 → 问题显影 → 弱测量与物理接近 → 承接与转交 → 回流与自消解
```

| # | 协议 | 职责 | 核心约束 |
|---|------|------|----------|
| 1 | 场域探测 | 评估问题场是否值得进入 | 不评人，只评场 |
| 2 | 问题显影 | 命名问题、暴露断口、降低伪清晰 | 不替代诊断下最终判词 |
| 3 | 弱测量与接近 | 自测、样本提交、低阻接近（注：PFM将LSM"弱测量"操作化为接引协议） | 不冒充正式诊断 |
| 4 | 承接与转交 | 说明边界、组织进入、转交下游 | 不直接许诺高影响结果 |
| 5 | 回流与消解 | 记录经验、退出、转研究入口 | 不把私有经验直接升格为法源 |

## 固定边界

PFM **不负责**：
- 不重新发明本体论（回 DM）
- 不自建桥接框架（回 ASTO）
- 不裁定制度正当性或文明优劣（回 ARBT）
- 不替代 COP 做风险分流
- 不单独裁定高影响责任问题（回 TAT）
- 不发明验证门禁与封存逻辑（回 ODD）

PFM **不输出**：临床诊断、治疗方案、法律/医疗/投资承诺、人格定性、TAT 责任阈值决定、ODD gate/seal/rollback、`R=1` 或无限兜底承诺。

## 跨层接口

| 方向 | 接口内容 |
|------|----------|
| → COP | 问题场编码 + 弱信号；COP 判断风险分流 |
| → LSM | 低风险时 LSM 帮助判断责任流向 |
| → TAT | 高影响责任转介 |
| → ODD | 前线承诺压成可审计对象 |
| → RT6 | 低风险操作执行入口 |
| → WSH | 生活场景复盘回正 |

### PFM -> WSH 最小转交包

PFM 只能在低风险、非临床、生活复盘 / 个人偏移回看场景中生成：

```text
pfm_to_wsh_handoff_packet
```

该包从 PFM-min 七字段派生，至少保留 `issue_field / pressure_bearer / constraint / weak_signal / boundary_notice / stop_condition`，并可增加 `life_dimension_hint / target_core_hint / deviation_snapshot_hint / referral_chain`。它只表示“问题已显影，可交给 WSH 判断是否进入归己复盘”，不表示 WSH 已接收、归己度 G 已成立或调适建议已成立。详细格式以 `AA-母版源层/PFM.305-转接矩阵.md` 为准。

## 高影响强制转介条件

出现以下任一情况，PFM **不得**单独推进，必须引入 TAT 与 ODD：

1. 涉及重大金钱损失或重大组织后果
2. 涉及合规、法律、公共风险
3. 涉及强责任承诺、补偿条款或闭门裁决
4. 涉及需要封存、审计、回滚的正式承诺

## ASTO 1-5-6-7-1 映射

```
1 当前混沌统一面
→ 5 显影五格（声/场/压/约/交）
→ 6 显影对话六步（收声/定场/定压/定约/成形/转交）
→ 7 七类显不出来断点（无声/错场/错人/伪清/越界/无接/断交）
→ 1 新的问题统一面
```

| ASTO 位点 | PFM 解释 |
|---|---|
| 1 当前混沌面 | 问题还说不清，只能看到弱信号、模糊不适或结构损耗 |
| 5 显影五格 | 声音、问题场、承压者、约束、转交口必须逐一显影 |
| 6 显影六步 | 收声、定场、定压、定约、成形、转交 |
| 7 七断点 | 无声、错场、错人、伪清、越界、无接、断交 |
| 1 新问题统一面 | 输出 `structured_problem_intake`，进入承接、转交或停止 |

## 不可替代问题与外部比较

> 详见 `CC-独立理论层/PFM.313-不可替代问题与外部比较.md`

### PFM 解决什么独特问题

> **一个人带着模糊的张力来到内容或产品面前——他怎么从"说不清自己怎么了"走到"问题被看见、被命名、愿意往前走一步"？**

传统体系分两路：市场营销回答"怎么吸引他"，用户体验回答"怎么让他用得顺"。两者都没有回答中间那片真空——**"他带着问题来了，但问题还说不清，怎么接住？"**

### 核心比较矩阵

| 维度 | PFM | 市场营销 | UX | 客服 |
|------|-----|----------|-----|------|
| 核心问题 | 问题怎么被看见 | 怎么吸引 | 怎么好用 | 怎么解决已有描述 |
| 输入状态 | 模糊、说不清 | 潜在兴趣 | 有行为意图 | 已明确问题 |
| 输出 | structured_problem_intake | 转化/线索 | 可用性 | 解决/记录 |
| 边界承接 | 内置 | 无 | 无 | 无 |

## 失败线总声明

> 详见 `CC-独立理论层/PFM.315-失败线总声明.md`

### 显影前提失效

| 编号 | 失效条件 | 后果 |
|------|---------|------|
| F1 | 问题无法被显影（用户无能力或不愿描述） | PFM 无法启动 |
| F2 | 显影形式无法被用户接收 | 问题被看见但被拒绝 |
| F3 | 不存在可承接的下游 | 问题显影后无处可去 |

### 五协议各自失效

| 协议 | 失效条件 |
|------|---------|
| 场域探测 | 场评分与后续显影成功无相关性 |
| 问题显影 | "伪清晰"率 > 50%（命名的问题不是真问题） |
| 弱测量与接近 | 自测工具产生系统性误导 |
| 承接与转交 | 下游承接率持续为 0 |
| 回流与消解 | 经验回流从未触发任何协议更新 |
